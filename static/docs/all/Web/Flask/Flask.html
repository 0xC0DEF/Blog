python을 선택한 이유

c++을 정말 좋아하는 나로써 python은 굉장히 생소한 언어다. 그럼에도 c++에 비해 python이 가지는 장점이 여럿 있기 때문에 python을 사용하기로 했다. 일단 c/c++로도 서버를 만들 수 있기는 하다. 성능이 굉장히 좋을테지만 단순한 블로깅 기능정도만 필요한 나로써는 오버엔지니어링(?)이다. 또한 백준문제를 풀면서, 특히 fft를 짜면서 c++과 대조되는 python 코드의 간결함을 자주 체험하기는 했다. python의 경우 어떤걸 개발하던지 자료가 많기 때문에 배워두면 앞으로도 도움이 많이 될거라는 생각에 python을 선택하게 되었다.

django에서 flask로

첫 시도는 django였다. 프로젝트를 만들면 뭔가 막 생기고 관리페이지도 생기고 하는데, 뭐가 뭔지 하나도 모르겠다. 책이 있는데도 말이다. 글의 카테고리가 트리구조를 형성하기 때문에, django template에서 재귀함수를 통해 트리를 순회하며 카테고리html코드를 만들어야 했는데, 명시적인 재귀함수는 없고 뭔가 이상한 트릭을 써서 해결하기 때문에 코드가 더러웠다. 결국 얼마 못가서 포기했다.

반면에 flask의 jinja2 템플릿은 재귀함수(매크로)를 지원하기 때문에 코드가 굉장히 깔끔하게 나왔다. 또한 기본코드가 10줄도 안되서 한번에 이해할 수 있었으며, 필요한 기능이 생길때마다 검색하여 추가하며 개발하고 있는 중인데, 굉장히 성공적이다! Flask의 micro framework 철학이 굉장히 마음에 들었다.
