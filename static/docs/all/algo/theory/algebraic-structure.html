Monoid

Segment Tree는 2가지 모노이드를 필요로 하는데, 나는 Q(Query Monoid)와 U(Update Monoid)라고 부르겠다. Lazy Update의 경우, 업데이트할 노드를 node, 업데이트값을 x라 하면 U.f( node.val, Q.f(...Q.f(id,x)...,x) )형태의 합성함수를 빠르게 계산해야 하는데, Q.fn(x,cnt)형태의 함수를 빠르게 계산할 수 있으면 된다. fn의 몇가지 예시: f=id+2는 fn=id+(2*cnt), f=min(id,x)=x는 fn=min(id,x)=x(그대로다) 등등.
